# Mobile Responsiveness Audit (2026-01-19)

## Core Architectural Issue
The application is architected as a horizontal scrolling experience driven by vertical scroll length (`HorizontalScrollContainer`). This relies on a "ghost" div of height `950vh` and transforms the content container horizontally.

**Impact:** On mobile, this feels unnatural and requires `vw` based sizing which breaks when the viewport width is small (portrait mode).

## Section Analysis

### 1. `App.tsx` & `HorizontalScrollContainer.tsx`
- **Issue:** `lg:hidden` overlay blocks mobile users.
- **Issue:** `HorizontalScrollContainer` forces horizontal layout `flex w-max` and uses fixed `h-[950vh]`.
- **Solution:** 
    - Remove overlay.
    - Conditionally render `HorizontalScrollContainer` logic. On mobile, use a native vertical scroll layout (`flex-col`, `overflow-y-auto`, normal height).

### 2. `TitleSection.tsx`
- **Issue:** `w-screen`.
- **Issue:** `text-[18vw]` is optimized for landscape. On portrait, this is too small.
- **Solution:** 
    - Use `w-full` on mobile.
    - Use `text-6xl` or similar for mobile, `md:text-[18vw]`.

### 3. `IntroSection.tsx`
- **Issue:** `w-[70vw]`. This wastes 30% of space on mobile if kept.
- **Solution:** Change to `w-full` on mobile (`lg:w-[70vw]`).

### 4. `BoltSection.tsx`
- **Issue:** `w-screen`. Content `w-[80%]`.
- **Solution:** `w-full`. `w-[80%]` is probably fine but might need `w-[90%]` on mobile.

### 5. `DukaanSection.tsx`
- **Issue:** `w-max`. Horizontal list of videos.
- **Issue:** Video cards are `w-[25vw]` or `w-[50vw]`.
- **Solution:** 
    - Switch container to `flex-col` or a grid.
    - Change video card widths to `w-full` or `aspect-ratio` based sizing.

## Action Items
1. Create a `MobileLayout` wrapper or modify `HorizontalScrollContainer` to accept a `isMobile` prop/logic.
2. Refactor all sections to use `className` props or internal responsive classes (`lg:w-[...]`, `w-full`).
